    Introduction

    Hey peeps, today we are going to understand slices in Golang. A slice is basically a dynamic array. Let’s see go’s definition of slices.

    A slice is a dynamically sized, flexible view into the elements of an array. They are like references to arrays. Changing the elements of a slice modifies the corresponding elements of its underlying array. Other slices that share the same underlying array will see those changes.

    ok, so slices are just pointing to the underlying array with some capacity.

    Creating Slices

    Let’s see how we can create slices,

    • Composite Literals
      • slice := []int{1, 2, 3, 4, 5}
         
    • make
      • slice := make([]int, 5) //This will have 5 ints with 0 value.
         
    • new
      • slice := new([]int) //will allocate memory and return the pointer, slice will have length and capacity as 0.
         

    These are few ways we can create slices. We have used int in the example but we can create slices of any type.

    Let us understand what is happening here, we have got 4 different addresses in the output so 4 new backing arrays are present. 

    Remember: In Go, everything is pass by value

    When updateIndex function is called the slice header is copied and passed to v, so v is a copy of the header. When we update the 0th index we actually get the underlying address so it gets updated, but when we do append, if the slice is grown then a new backing array is allocated and the new slice header is returned.

    Since we are doing append on a copy of the header, it won’t update the actual slice. Because of this the One, Two, Three are present inside the function on Line 2 but not on line 3. Again on line 5, we see the same thing happening.

    Again on output Line 7, We can see  otherSlice has grown because of append, and its address is changed.

    Now, what if we wanted the append function to work inside the function? We simply use *[]string as a parameter of the function, so instead of the header copy the actual header is sent, and updating works. You can test that on the Playground.

    Conclusion

    So to conclude, If we are passing a slice to a function and we want to update just the index of a slice then we don’t need a pointer but if we plan to do append operations then we need a pointer.
