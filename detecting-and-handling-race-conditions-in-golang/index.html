    What is a race condition ?

    A race condition basically means that there is a data race. i.e some resource is accessed by multiple threads or goroutines. In Go, a resource could mean a struct, map, variable, etc, and most of them are not concurrency/thread-safe unless explicitly stated.

    A thread-safe example would be some database connection that can be created as a global variable and can be used anywhere concurrently.

    Fixing Race Conditions

    So now what ? after detecting the race condition how can we fix it? sync package to the rescue.

    Sync package provides us with mutexes that can be used to synchronize the code.

    We can create the mutex like this – mx := &sync.Mutex{} . Mutex provides 2 methods, Lock and Unlock . These methods will lock the code snippets in between so basically the code between them will always be executed in sync for this mutex.
