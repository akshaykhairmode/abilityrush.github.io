    Introduction

    Unit tests mean writing code to test a set of code or functions with predefined input and output. This test code does not have any business logic written in it, but takes input and expected output and compares them, and says success or failure based on the result.

    For example, if we have a function createFile then we can write a unit test to verify if the file gets created by calling the createFile function inside our test function.

    How to write unit tests

    Go provides a testing package in the standard library that helps us writing unit tests. There are a few rules for writing with this package,

    1. The function name should start from Test (T should be capitalized) 
    2. It should have only 1 parameter *testing.T
    3. Methods like error, fatal, fail, etc must be used if we want to fail the test else the test will be successful.
    4. The test file should be separate which will be named as _test.go
