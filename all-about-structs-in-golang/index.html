    Introduction

    Structs are basically a collection of fields of any type. We use it to logically group data, For example, if we are creating a program where we have to work with shops then our fields would be name, address, typ, phone, customers then we can create a struct shop that will have these details.

    This is the most common user-defined data structure.

    Syntax for structs

    Considering the example of a shop we can create a struct like below,

    type shop struct {
        Name                          string
        Address                      string
        Typ                              string
        Phone                          int
        AverageCustomers *int
    }

    For average customers, we are using a pointer of int. Let’s create few shops,

    shop1 := shop{“Shop1”, “Canada”, “Grocery”, 1111111111, new(int)}

    shop2 := shop{Name: “Shop2”, Address: “US”, Typ: “Pharmacy”, Phone: 1111111111, AverageCustomers: new(int)}

    shop3 := new(shop)
    shop3.Name = “Shop3”
    shop3.Address = “Japan”
    shop3.Typ = “Restaurant”
    shop3.Phone = 1111111111
    shop3.AverageCustomers = new(int)

    We are using struct literals for creating structs. For shop1 we are passing the data directly. It will follow the declaration order of the struct. Shop2 has field names present so the order is not important in this case. shop3 is created using new which returns a pointer so we are setting the fields after that.

    Note – If we add a field name for some key in shop1 then the compiler will throw an error as we cannot have both together.

    Let’s create a printShop function that will print the shops.

    func printShop(s …shop) {

        for _, v := range s {
            fmt.Printf(“Printing for : %v —- %+v”, v.Name, v)
            fmt.Printf(” – AverageCustomers Value – %v \n”, *v.AverageCustomers)
        }

    }

    Now let’s make some functions which will update the shop,

    func updatePhone(s shop, v int) {
        s.Phone = v
    }

    func updateCustomers(s shop, v int) {
        *s.AverageCustomers = v
    }

    Finally, let’s write some code to run our functions,

    fmt.Println(“Printing Initial Values…”)
    printShop(shop1, shop2, *shop3)
    fmt.Println(“Updating Customer Values…”)
    updateCustomers(shop1, 5)
    updateCustomers(shop2, 10)
    updateCustomers(*shop3, 15)
    fmt.Println(“Printing Values After Customer Update…”)
    printShop(shop1, shop2, *shop3)
    fmt.Println(“Updating Phone Values…”)
    updatePhone(shop1, 000000000)
    updatePhone(shop2, 000000000)
    updatePhone(*shop3, 000000000)
    printShop(shop1, shop2, *shop3)

    After running our code, we got the below output,

    Printing for : Shop1 —- {Name:Shop1 Address:Canada Typ:Grocery Phone:1111111111 AverageCustomers:0xc000012078} – AverageCustomers Value – 0

    We can see that the memory address is being printed for AverageCustomers. Also, AverageCustomers value is getting updated even when we are passing a copy, This is because the address remains the same even if a copy is passed.

    We are also trying to update the phone but it’s not working because the address is not in the picture. so it’s not updating the original value.

    JSON and Structs

    So now we know how structs work. Let’s see how JSON and other libraries work with struct tags.

    Let’s modify our struct,

    type shop struct {
        Name             string `json:”shopName”`
        Address          string `json:”shopAddr”`
        Typ                 string `json:”type”`
        Phone            int    `json:”-“`
        AverageCustomers *int   `json:”avgCust”`
    }

    We have added some JSON tags in this struct. Now JSON package can read these and can use it when encoding or decoding json.

    Adding some more code below,

    var js []byte
    js, _ = json.Marshal(shop1)
    fmt.Printf(“Json Marshal Shop1 : %s\n”, js)

    js, _ = json.Marshal(shop2)
    fmt.Printf(“Json Marshal Shop2 : %s\n”, js)

    js, _ = json.Marshal(shop3)
    fmt.Printf(“Json Marshal Shop3 : %s\n”, js)

    And our output is,

    Json Marshal Shop1 : {“shopName”:”Shop1″,”shopAddr”:”Canada”,”type”:”Grocery”,”avgCust”:5}
    Json Marshal Shop2 : {“shopName”:”Shop2″,”shopAddr”:”US”,”type”:”Pharmacy”,”avgCust”:10}
    Json Marshal Shop3 : {“shopName”:”Shop3″,”shopAddr”:”Japan”,”type”:”Restaurant”,”avgCust”:15}

    The point to note is that even though shop3 is a pointer JSON marshall still works as expected.

    Empty structs

    So what are empty structs? They are like normal structs only but do not have any fields attached to them. ok so why would we use these? Well because they consume zero space to exist.

    To understand better, let’s make some slices of length 50 million.

    _ = make([]int, size)

    _ = make([]float64, size)

    _ = make([]complex128, size)

    _ = make([]bool, size)

    _ = make([]byte, size)

    _ = make([]struct{}, size)

    Now we check the allocation each of these do so we create a function that will print allocations and execute the GC. We call this function after every make line.

    func printMemUsage(v string) {

        defer runtime.GC()

        var m runtime.MemStats
        runtime.ReadMemStats(&m)

        fmt.Print(v + fmt.Sprintf(” – Memory Allocated : %v MB\n”, m.Alloc/1024/1024))

    }

    We also need to stop the GC to run automatically so we use this line – debug.SetGCPercent(-1)

    So our main function looks like this,

    func main() {

        debug.SetGCPercent(-1)

        _ = make([]int, size)
        printMemUsage(“Using Int”)

        _ = make([]float64, size)
        printMemUsage(“Using Float”)

        _ = make([]complex128, size)
        printMemUsage(“Using Complex”)

        _ = make([]bool, size)
        printMemUsage(“Using Bool”)

        _ = make([]byte, size)
        printMemUsage(“Using Byte”)

        _ = make([]struct{}, size)
        printMemUsage(“Using Empty Structs”)
    }

    If we run our code we get the below output,

    akshay@AKSHAY-PC:~/learning-go/structs$ go run empty_struct.go
    Using Int – Memory Allocated : 381 MB
    Using Float – Memory Allocated : 381 MB
    Using Complex – Memory Allocated : 763 MB
    Using Bool – Memory Allocated : 47 MB
    Using Byte – Memory Allocated : 47 MB
    Using Empty Structs – Memory Allocated : 0 MB

    If we see the allocations, empty structs have 0MB allocated. Now let’s say we want to create a set. we can use a map for creating it but the value in a set is useless so we can use map[int]struct{}so if our map has millions of elements then our memory used will only be for the int keys.

    Since these behave the same as a normal struct we can also attach methods to these structs.

    Conclusion

    So to conclude, We have 2 types of structs, a normal struct, and an empty struct. normal structs use some memory while empty structs do not. Methods can be attached to both of them.

    Any type of field can be used inside when creating structs.
