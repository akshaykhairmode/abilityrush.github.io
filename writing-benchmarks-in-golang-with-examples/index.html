    Introduction

    Hey guys, based on the title you must have guessed what we are going to do today. But first, let’s understand why do we need benchmarks?

    We need benchmarks because they tell us what performance we can expect with the current code and if we can improve it. We compare our benchmark result with different functions or branches and choose the fastest.

    So today we will write a function to write to file 100000 lines, each line having the iteration number, and see how we can make it faster.

    NOTE : Running benchmark can give different results for different iterations

    Writing to File (buffered)

    Let’s write a buffered writer function, 

    We are using bufio package to write in a buffered manner. This will reduce the number of IO calls. So basically writeString will keep writing in memory till that buffer is full. Once full it’s written to the file. We are flushing at the end in case any data remained in the buffer.

    NOTE – The default buffer size is 4KB. We can use bufio.NewWriterSize() for a custom buffer size.

    Let’s run the benchmark command again, We see the above output. We can see that our old function is still running 5 iterations only but our buffered writer has run for 93 iterations. The improvement is around 19x.

    Other Benchmark Options

    -benchmem

    This option also gives us the memory allocation which is happening. We can run the benchmark like go test -bench=. -benchmem. This will give us the below output.

    We can see the last column is showing 199749 allocs/op this is the allocated memory per operation. Looks like the allocation is the same for both functions.

    Using pprof

    Pprof is basically the profiling tool/package provided in the standard library. We can use pprof as a web server or withgo tool pprof. We will look into the go tool pprof part today.

    First, let’s create the profiles, We will be creating 3 files which will contain profiles for the following -cpuprofile,-memprofile,-trace

    Our command will be – go test -bench=. -benchmem -cpuprofile=cpu.out -memprofile=mem.out -trace=trace.out

    The output of this command will be like below,

    Not different from our previous command but the files are created. see below.

    Let’s use go tool pprof -http :8080 mem.out command. This will start an HTTP server on port 8080 and open URL – :8080/ui/ which will display how memory is getting allocated.

    Here we can see our Sprintf method is using a lot of memory. So we are going to use strconv.Itoa() instead. Let’s see how our benchmark changes after removing fmt.Sprintf().
