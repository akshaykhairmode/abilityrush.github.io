    Introduction

    Hey folks, today we will see the Context package in Go standard library. We usually use Context package to get deadlines to cancel our processing.

    Context has a collapsing effect, so if multiple calls use context with some deadline, the context ending at the end can help close all the middle children. More information is here.

    We are going to see the below ways to use Context along with use cases. This post is on the bigger side, feel free to jump to sections with the below anchors.

    1. With Deadline
    2. With Timeout
    3. With Cancel
    4. With Value

    Other real-world use cases we will see are related to file reading,

    1. Stop reading of file if the context is complete/canceled
    2. How the canceling works with an HTTP request.

    Creating a context is simple, We will use the below line to create it.

    ctx := context.Background()

    Context with Deadline

    Let’s look at the below code,  We are passing a context to our WithDeadline function. Here we are getting a time which is after 5 seconds and passing it to the context.WithDeadline method.

    We are calling the method ctx.Done() that returns us a channel that we consume, as we already know reading from the channel is a blocking operation but in the case of for select loops it will move to the default case.

    func WithDeadline(ctx context.Context) {

	log.Println("WithDeadline")

	now := time.Now()
	log.Println("Start time is", now)

	after5seconds := now.Add(5 * time.Second)
	log.Println("Deadline is", after5seconds)

	ctx, cancel := context.WithDeadline(ctx, after5seconds)
	defer cancel() //Release the context

	for {
		select { //Default would be last priority here.
		case <-ctx.Done():
			log.Println("Exiting function WithDeadline")
			return
		default:
			time.Sleep(500 * time.Millisecond)
			log.Printf("This line will keep printing every 500ms")
		}
	}

}

    Context with Timeout

    WithTimeout is similar to our deadline except giving a specific moment in time, we give a relative time, i.e 5 seconds or 10 minutes, etc. Our code and output are below,

    func WithTimeout(ctx context.Context) {

	log.Println("WithTimeout")

	ctx, cancel := context.WithTimeout(ctx, 5*time.Second) //Will wait for 5 seconds
	defer cancel()                                         //Release the context

	for {
		select {
		case <-ctx.Done():
			log.Println("Exiting function WithTimeout")
			return
		default:
			time.Sleep(500 * time.Millisecond)
			log.Printf("This line will keep printing every 500ms")
		}
	}
}

    Context with Cancel

    In our earlier code examples, you must be thinking about what is the cancel function that’s being returned.

    This cancel and our earlier cancels are the same functions that can be called to cancel a context manually. So for our example, we will create a context with cancel and cancel it manually in 2 seconds.

    Our function will return in 2 seconds in this case. Let’s see the code and the output.

    func WithCancel(ctx context.Context) {

	log.Println("WithCancel")

	ctx, cancel := context.WithCancel(ctx) //returns a cancel func which can be called separately to send signal to done channel
	defer cancel()                         //Release the context

	go func() { //in 2 seconds call the cancel function.
		time.Sleep(2 * time.Second)
		cancel()
	}()

	for {
		select {
		case <-ctx.Done():
			log.Println("Exiting function WithCancel")
			return
		default:
			time.Sleep(500 * time.Millisecond)
			log.Printf("This line will keep printing every 500ms till 2 second passes")
		}
	}
}

    Context with Value

    Coming to our last context, WithValue gives us a way to store some key-value pairs inside the context. This key-value pair is passed on ahead and can be modified based on the key type.

    Our key and value are both empty interface types, so we can pass any values there. We should avoid passing string values to keys as they can collide and override.

    I have created a demo package that creates some context based on the different keys to demonstrate how the context Get and Modify operations work. The code is as below, please check the inline comments.

    The Demo Package

    package demo

import "context"

type myPrivateKey string

type MyExportedKey string

//NOTE :: All our keys will have the same string value that is ID.

//This is a normal string key, can be used anyone by outside the package
var ctxKey = "ID"

//This key is of type myPrivateKey and the type is only accessible by this package.
var ctxPrivateKey myPrivateKey = "ID"

//This key is of type MyExportedKey and the type is accessible to whoever imports this package
var ctxExportedKey MyExportedKey = "ID"

//Some value which I may not want others to change.
var valueThatShouldNotChange = "SOME_SECRET_STRING"

//returns a context which has ID key but the datatype is not exported.
func Private() context.Context {
	ctx := context.Background()
	dctx := context.WithValue(ctx, "NAME", "John") //Default value
	return context.WithValue(dctx, ctxPrivateKey, valueThatShouldNotChange)
}

//Returns a context which has ID key but the datatype is string that anyone can create.
func Public() context.Context {
	ctx := context.Background()
	dctx := context.WithValue(ctx, "NAME", "John") //Default value
	return context.WithValue(dctx, ctxKey, valueThatShouldNotChange)
}

//Returns a context which has ID key but the datatype is exported.
func PublicWithExportedType() context.Context {
	ctx := context.Background()
	dctx := context.WithValue(ctx, "NAME", "John") //Default value
	return context.WithValue(dctx, ctxExportedKey, valueThatShouldNotChange)
}

    Let’s now write some code that will modify and print the contexts to understand how access and modification work.

    func WithValue(ctx context.Context) {

	//Add ID key to passed context. Replaces the old ID if existed.
	ctx = context.WithValue(ctx, "ID", "SOME-STRING")

	//Check comments in demo file.
	privateCtx := demo.Private()
	publicCtx := demo.Public()
	exportedPublicCtx := demo.PublicWithExportedType()

	//Trying modification on different context types.
	modifiedPrivateCtx := context.WithValue(privateCtx, "ID", "MY-MODIFIED-STRING")
	modifiedPublicCtx := context.WithValue(publicCtx, "ID", "MY-MODIFIED-STRING")
	modifiedExportedCtx := context.WithValue(exportedPublicCtx, demo.MyExportedKey("ID"), "MY-MODIFIED-STRING")

	//Maintain to print easily all of them.
	print := map[context.Context]string{
		ctx:                 "ctx",
		privateCtx:          "privateCtx",
		publicCtx:           "publicCtx",
		exportedPublicCtx:   "exportedPublicCtx",
		modifiedPrivateCtx:  "modifiedPrivateCtx",
		modifiedPublicCtx:   "modifiedPublicCtx",
		modifiedExportedCtx: "modifiedExportedCtx",
	}

	for ctx, what := range print {
		printContext(ctx, what)
	}
}

//Prints the context values
func printContext(ctx context.Context, which string) {
	log.Printf("%s ::: ID is %v, NAME is %v, exported ID is %v", which, ctx.Value("ID"), ctx.Value("NAME"), ctx.Value(demo.MyExportedKey("ID")))
}

    Let us understand the output now. We are trying to get the values of 3 keys here.

    • ID (datatype string)
    • NAME (datatype string)
    • ID (datatype MyExportedKey)

    Coming to each line,

    ctx ::: ID is SOME-STRING, NAME is <nil>, exported ID is <nil> – We see ID here is set to SOME-STRING which we set at the start. NAME is nil as we are setting it in demo package context functions. exported ID is nil as we did not try setting it for this context.

    privateCtx ::: ID is <nil>, NAME is John, exported ID is <nil> – This context came from demo package private type. ID is nil as string ID is not present. Name is john as expected. exported ID is nil as we did not set it. Also we dont have a way to access the ID key which was set here. If we want to access that key, we need to create a getter function inside the demo package which can extract it.

    publicCtx ::: ID is SOME_SECRET_STRING, NAME is John, exported ID is <nil> – In case of public we used string to set the key so its also accessible outside of package. Name as expected. exported ID was not set.

    exportedPublicCtx ::: ID is <nil>, NAME is John, exported ID is SOME_SECRET_STRING – Here ID is nil because string ID was not set in demo package. Name is as expected. We have got the exported ID value as we can access the type of that key.

    modifiedPrivateCtx ::: ID is MY-MODIFIED-STRING, NAME is John, exported ID is <nil> – We tried modifying private context. We can see that ID is modified here, but note here that this is the string ID. our actual secret ID is still in the context but not accessible here. Name, Exported ID is as expected.

    modifiedPublicCtx ::: ID is MY-MODIFIED-STRING, NAME is John, exported ID is <nil> – Here the ID is overwritten because the key was string. Name, Exported ID as expected.

    modifiedExportedCtx ::: ID is <nil>, NAME is John, exported ID is MY-MODIFIED-STRING – String ID was not set here. Name is as expected. We can see that we have modified the exported ID here because we have access to the actual data type.

    Finally, Whenever we are creating a Context with value we shouldn’t use built-in data types but custom types. Empty struct types are commonly used. For example, converting our type myPrivateKey string to type myPrivateKey struct{}

    Cancel file read operation with Context

    Let us first create a file having each alphabet on every line. We will now create a function for reading the file. Our function will do below steps,

    1. Open File
    2. Get scanner from bufio package.
    3. Infinitely check for context done or file reading to complete.
    4. Sleep for some time after every read operation to simulate a big file.
    5. return whatever string is created till now

    Below is our code, check the inline comments, and let’s see the output.

    func fileReadExample(ctx context.Context) (string, error) {

	f, err := os.Open("data.txt")
	if err != nil {
		return "", err
	}

	s := bufio.NewScanner(f)
	out := strings.Builder{}

t:

	for {

		select {
		//If context is done then we wull return from this function with error.
		//We can choose if we want to return the data read till now or return empty string.
		case <-ctx.Done():
			return out.String() + "\n", fmt.Errorf("ctx cancelled")
		default:
			//Till my context is done default will be called to scan every line an append to our string builder.
			if s.Scan() {
				log.Printf("Appending %s", s.Text())
				out.WriteString(s.Text())
				time.Sleep(500 * time.Millisecond)
			} else {
				//When scan returns false, it means that the file has reached the end
				//so we break the loop with a label.
				break t
			}
		}
	}

	return out.String() + "\n", nil
}

    To the right is the output with context timeout as 10 seconds.

    We can see that it has partially returned the string and canceled as soon as the 10 seconds elapsed.

    This is our output with 20 seconds context timeout.

    We can see that we have got the full string without any error.

    Context cancel with HTTP

    For demonstrating cancellation with HTTP, We will create an HTTP server and on the root handler, we will call our fileReadExample function passing the request context and see how it works when the user cancels the request.

    Let’s see our code, and check the inline comments and our output.

    func startHttpServer() {
	s := http.Server{
		Addr: ":8000", //Start server on this port
		Handler: http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) { //On base path execute below code.
			data, err := fileReadExample(r.Context()) //Pass the http request context
			if err != nil {                           //Context cancels or any other erros will come here.
				log.Println(err)
				w.WriteHeader(500) //Give 500 status code if 500 error
				w.Write([]byte(data))
				return
			}

			w.WriteHeader(200) //200 status code if success
			w.Write([]byte(data))
			log.Println("Request Complete")
		}),
	}

	log.Println("Starting server on port 8000")
	log.Println(s.ListenAndServe()) //Start the http server

    In our output, we can see that I did 2 requests,

    • The first one was canceled by me and our processing stopped the function as seen by the logs on the left.
    • The second one was not canceled and completed giving us the output of all the alphabets.
