    Introduction

    Hi Guys, In this post, we are going to use a custom handler for your HTTP server. To implement it we are going to create a simple user addition application.

    We are going to do the below,

    1. Create 3 endpoints, get_user, add_user and delete_user. 
    2. Using our custom JSON handler which will have the ServeHTTP method.
    3. Create a users struct that will have a map of user data and RW mutex.
    4. Run the application and see the output.

    Declaring Types and Variables

    Let’s create a details struct that will hold the user’s first name and last name.

    type details struct {
    FirstName string `json:"fname"`
    LastName  string `json:"lname"`
}

    Since our application will be JSON-based, let’s define a response for the API. The reason for using details as a pointer is that we don’t need it to come in the response if it’s nil. If we use a copy instead of a pointer it will come in the response.

    type response struct {
    Status     string   `json:"status"`
    Message    string   `json:"message"`
    Data       *details `json:"data,omitempty"`
    statusCode int
}

    We need a users struct that will hold all the data and mutex,

    type users struct {
    data   map[int]details
    currID int
    *sync.RWMutex
}

    Finally, we declare our handler struct, the handler just needs the ServeHTTP method so we are using an empty struct for it.

    type jsonHandler struct{}

    Defining ServeHTTP method

    In serveHTTP method we will handle our endpoints. The advantage of using a custom handler is that we can write a common code for our endpoints.

    For example, Since our application would be JSON based we want the content type to be application/json for all the requests and also if any other header needs to be passed.

    func (mh jsonHandler) ServeHTTP(rw http.ResponseWriter, r *http.Request) {

    rw.Header().Add("X-AbilityRush-Domain", "abilityrush.com")
    rw.Header().Add("Content-Type", "application/json")

    var resp response

    base := r.URL.Path

    fmt.Printf("base: %v\n", base)

    switch base {
    case "/get_user":
        id := r.URL.Query().Get("id")
        resp = getUser(id)
    case "/add_user":
        resp = addUser(r.Body)
    case "/delete_user":
        id := r.URL.Query().Get("id")
        resp = delUser(id)
    default:
        resp.statusCode = 404
        resp.Message = "404 Not Found"
        resp.Status = "failed"
    }

    respBytes, _ := json.Marshal(resp)

    rw.WriteHeader(resp.statusCode)
    rw.Write(respBytes)

}

    Looking at the ServeHTTP method, header and response code needs to be executed for every request. We can also see the endpoints defined are calling specific functions that return a response type which we marshall later and write the output to the responseWriter

    Define endpoint functions

    We will now define our addUser, delUser, getUser functions, But first let’s define our global userData variable which will be of type users.

    addUser

    var userData = users{data: make(map[int]details), RWMutex: &sync.RWMutex{}}

func addUser(r io.ReadCloser) response {

    var resp response

    data, err := ioutil.ReadAll(r)
    if err != nil {
        fmt.Println(err)
        resp.statusCode = 500
        resp.Status = "failed"
        resp.Message = "Some Error Occured"
        return resp
    }

    ud := details{}

    if err := json.Unmarshal(data, &ud); err != nil {
        fmt.Println(err)
        resp.statusCode = 400
        resp.Status = "failed"
        resp.Message = "Invalid JSON passed"
        return resp
    }

    userData.Lock()
    userData.currID += 1
    userData.data[userData.currID] = ud
    userData.Unlock()

    return response{statusCode: 200, Status: "success", Message: "User Added Successfully, UID : " + strconv.Itoa(userData.currID)}

}

    We can see in add user we are getting the JSON from the caller and doing an unmarshal. If it’s a success then we need to add the user. We lock the struct, increment the currID and assign it to the map. 

    getUser

    func getUser(idStr string) response {

    fmt.Println("Got ID : ", idStr)

    var resp response

    id, err := strconv.Atoi(idStr)
    if err != nil {
        fmt.Println(err)
        return strToIntResponse
    }

    resp.statusCode = 200
    resp.Status = "success"

    userData.RLock()
    ud, ok := userData.data[id]
    userData.RUnlock()

    if !ok {
        resp.Message = "User does not exist"
        return resp
    }

    resp.Message = "User Found"
    resp.Data = &ud
    return resp
}

    In this function, we are finding the data from the map and returning the appropriate response to the caller. We are using Rlock while getting the data because we are doing just a read operation. Then we assign the details to the data field.

    delUser

    func delUser(idStr string) response {

    fmt.Println("Got ID : ", idStr)

    id, err := strconv.Atoi(idStr)
    if err != nil {
        fmt.Println(err)
        return strToIntResponse
    }

    var resp response

    userData.Lock()
    delete(userData.data, id)
    userData.Unlock()

    resp.statusCode = 200
    resp.Message = "Success"
    resp.Status = "User Deleted"
    return resp

}

    We can see the code from lines 13-15, we are deleting the userData. Quite a simple function.

    Note: We are not checking if the user exists before deleting.

    Output of our HTTP server

    Let’s run our application and do some curl calls to our endpoint. Since our responses do not have a newline (\n) we are doing an echo on terminal.

    • curl localhost:7000 && echo
    • curl localhost:7000/get_user && echo
    • curl localhost:7000/add_user –data ‘{“fname”:”John”,”lname”:”Doe”}’ && echo
    • curl localhost:7000/add_user –data ‘{“fname”:”Johnny”,”lname”:”Depp”}’ && echo
    • curl localhost:7000/get_user?id=2 && echo
    • curl localhost:7000/delete_user?id=1 && echo
    • curl localhost:7000/get_user?id=1 && echo
    • curl localhost:7000/get_user?id=2 && echo

    Conclusion

    We saw how we can create a custom handler which is useful when doing some common stuff like giving our JSON responses to all our requests adding headers and so on.
