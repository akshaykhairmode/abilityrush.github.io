    Introduction

    Hey Folks, after seeing download managers downloading files concurrently with multiple connections, I was quite curious about how it’s done.

    So we are going to make a small tool with Go for downloading files concurrently over HTTP. Understanding mutexes, wait groups, flags & goroutines is a prerequisite.

    This is a series of 4 posts which include memory management, progress bar, and resume functionality.

    1. This post
    2. Memory Management
    3. Progress bar implementation
    4. Pause & Resume

    How it works ?

    Before going to the code, Let’s see how it actually works. Download managers create multiple connections to the web server serving the file and download a chunk of the file and finally merge them together. 

    For this to work, the web server must support the Range header. The range header tells us that we can ask for a part of the file instead of the whole file by sending the range header to it and mentioning the bytes which we want to get. Quite simple.

    Code

    Let’s get to the coding part now,

    For part 1, our tool will have below features,

    • Option to set the concurrency for downloading the file.
    • Option to change the download path of the file.

    Since we are downloading a file on our machine, we are basically summoning the file xD, so I am going to call the tool summon. 

    type summon struct {
	concurrency int
	uri         string
	chunks      map[int][]byte
	err         error
	opath       string
	*sync.Mutex
}

    This is our summon struct, 

    • concurrency – This is the number of connections which we want to use for downloading the file.
    • uri – This is the file url which will be passed.
    • chunks –  This will contain the actual data which was downloaded. We are using map here because finally we want to merge the data in proper order or it will create a corrupt file.
    • err – This represents if any error occured while downloading.
    • opath – This is the output file path passed.
    • We are using anonymous mutex as we need locking when updating map.

    Find if range is supported

    Our first job is to find out if the passed URLs web server supports the Range header. To do this we will do a request to the URL using the HTTP method HEAD. Using this method we will only get the headers and no body. From the headers returned we can check if the range header is supported. If we see the below image which has the head request, the server returns accept-ranges: bytes and also the content length at the bottom.

    We will create a getRangeDetails method which will tell us if the range is supported, We will pass the URL of the file to the function and it will return to us if the range is supported, the content length and error if any.

    //getRangeDetails returns ifRangeIsSupported,statuscode,error
func getRangeDetails(u string) (bool, int, error) {

	request, err := http.NewRequest("HEAD", u, strings.NewReader(""))
	if err != nil {
		return false, 0, fmt.Errorf("Error while creating request : %v", err)
	}

	sc, headers, _, err := doAPICall(request)
	if err != nil {
		return false, 0, fmt.Errorf("Error calling url : %v", err)
	}

	if sc != 200 && sc != 206 {
		return false, 0, fmt.Errorf("Did not get 200 or 206 response")
	}

	conLen := headers.Get("Content-Length")
	cl, err := strconv.Atoi(conLen)
	if err != nil {
		return false, 0, fmt.Errorf("Error Parsing content length : %v", err)
	}

	//Accept-Ranges: bytes
	if headers.Get("Accept-Ranges") == "bytes" {
		return true, cl, nil
	}

	return false, cl, nil

}

    How to download the chunk

    Now let’s create a downloadFileForRange function which will download the chunk and set it to the map. Here parameter u is url and r is the range of bytes which we want to download, index represents the order of the chunk. 

    Example – If the content length is 1000, and we have concurrency as 10 then, index 0 will have 1 – 100 bytes, similarly for index 1 we will have 101 – 200 bytes and so on.

    https://github.com/akshaykhairmode/summon/blob/v0.1.2/main.go#L187

    func (sum *summon) downloadFileForRange(wg *sync.WaitGroup, u, r string, index int) {

	if wg != nil {
		defer wg.Done()
	}

	request, err := http.NewRequest("GET", u, strings.NewReader(""))
	if err != nil {
		sum.err = err
		return
	}

	if r != "" {
		request.Header.Add("Range", "bytes="+r)
	}

	sc, _, data, err := doAPICall(request)
	if err != nil {
		sum.err = err
		return
	}

	//206 = Partial Content
	if sc != 200 && sc != 206 {
		sum.Lock()
		sum.err = fmt.Errorf("Did not get 20X status code, got : %v", sc)
		sum.Unlock()
		log.Println(sum.err)
		return
	}

	sum.Lock()
	sum.chunks[index] = append(sum.chunks[index], data...)
	sum.Unlock()
}

    Processors

    We will create 2 methods, processSingle and processMultiple, processSingle will be called when we get concurrency <= 1 or if the web server does not support range.

    Let’s explore the processMultiple method as it will execute in case of concurrency > 1. As we can see in the below image, we are passing the content length to the function,  we then create a split variable and create a range based on that which we are passing to the downloadFileForRange function, and at the end we are combining the chunks together.

     https://github.com/akshaykhairmode/summon/blob/v0.1.2/main.go#L119

    func (sum *summon) processMultiple(contentLength int) error {

	split := contentLength / sum.concurrency

	wg := &sync.WaitGroup{}
	index := 0

	for i := 0; i < contentLength; i += split + 1 {
		j := i + split
		if j > contentLength {
			j = contentLength
		}

		//Initialize for each index or application will panic
		sum.chunks[index] = make([]byte, 0)
		wg.Add(1)
		go sum.downloadFileForRange(wg, sum.uri, strconv.Itoa(i)+"-"+strconv.Itoa(j), index)
		index++
	}

	wg.Wait()

	if sum.err != nil {
		return sum.err
	}

	return sum.combineChunks()
}

    Finally, we will create the combineChunks function which will create the file and write the chunks to the file. We will not use a for-range loop because we want to combine the bytes in an ordered manner.

    https://github.com/akshaykhairmode/summon/blob/v0.1.2/main.go#L149

    The Full code for this is on GitHub Repo – https://github.com/akshaykhairmode/summon 
    Tag – v0.1.2

    //combineChunks will combine the chunks in ordered fashion starting from 1
func (sum *summon) combineChunks() error {

	var fname string

	if sum.opath != "" {
		fname = sum.opath
	} else {
		currDir, err := os.Getwd()
		if err != nil {
			return fmt.Errorf("Error while getting pwd : %v", err)
		}
		fname = currDir + "/" + filepath.Base(sum.uri)
	}

	out, err := os.Create(fname)
	defer out.Close()

	if err != nil {
		return fmt.Errorf("Error while creating file : %v", err)
	}

	buf := bytes.NewBuffer(nil)
	//Not using for range because it does not gurantee ordered iteration
	for i := 0; i < len(sum.chunks); i++ {
		buf.Write(sum.chunks[i])
	}

	l, err := buf.WriteTo(out)
	if err != nil {
		return fmt.Errorf("Error while writing to file : %v", err)
	}

	log.Printf("Wrote to File : %v, len : %v", fname, l)

	return nil
}
