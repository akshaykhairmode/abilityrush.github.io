    In part 2 we updated the code to be more memory efficient using files and io.Copy. Now we will try to implement a progress bar in the tool. This progress bar will show progress for each of the connections. We will also handle the user kill commands like ctrl+c.

    Let’s start. Full code available here

    Our updated struct is, We have added a stop channel which will hold an error. We have added a progress bar which is a map. The Key will contain the connection index and the value is a progress struct.

    type summon struct {
	concurrency   int               //No. of connections
	uri           string            //URL of the file we want to download
	chunks        map[int]*os.File  //Map of temporary files we are creating
	err           error             //used when error occurs inside a goroutine
	startTime     time.Time         //to track time took
	fileName      string            //name of the file we are downloading
	out           *os.File          //output / downloaded file
	progressBar   map[int]*progress //index => progress
	stop          chan error        //to handle stop signals from terminal
	*sync.RWMutex                   //mutex to lock the maps which accessing it concurrently
}

    Below is the progress struct, curr holds the number of bytes read for that connection and the total is self-explanatory.

    type progress struct {
	curr  int //curr is the current read till now
	total int //total bytes which we are supposed to read
}

    Updating Progress Bar

    We have updated our progress function to initialize the progress struct, We are going to update the curr after every read.

    sum.chunks[index] = f
sum.progressBar[index] = &progress{curr: 0, total: j - i}

    Let’s see the read body function which we have created, It will run in an infinite loop till we get an EOF from read. buf has a size of 500 bytes, so for every iteration of readBody we try to read 500 bytes though read may not fill it completely. Hence we check if it’s greater than 0 and write only the number of bytes actually read else the initial bytes will be written to the file also.

    And after that, we are updating the progress bar current with the total bytes read till now.

    func (sum *summon) readBody(response *http.Response, f io.Writer, buf []byte, readTotal *int, index int) error {

	r, err := response.Body.Read(buf)

	if r > 0 {
		f.Write(buf[:r])
	}

	if err != nil {
		return err
	}

	*readTotal += r

	sum.Lock()
	sum.progressBar[index].curr = *readTotal
	sum.Unlock()

	return nil
}

    Now let’s see our earlier getDataAndWriteToFile function, we have changed it as below, We are not using io.Copy here and reading in 500 byte iteration else we can’t find the progress. We will come back for the select case. We can see here when EOF is received then we are returning from this function.

    func (sum *summon) getDataAndWriteToFile(request *http.Request, f io.Writer, index int) (int, error) {

	client := http.Client{
		Timeout: 0,
	}

	response, err := client.Do(request)
	if err != nil {
		return response.StatusCode, fmt.Errorf("Error while doing request : %v", err)
	}
	defer response.Body.Close()

	//we make buffer of 500 bytes and try to read 500 bytes every iteration.
	var buf = make([]byte, 500)
	var readTotal int

	for {
		select {
		case cErr := <-sum.stop:
			return response.StatusCode, cErr
		default:
			err := sum.readBody(response, f, buf, &readTotal, index)
			if err == io.EOF {
				return response.StatusCode, nil
			}

			if err != nil {
				return response.StatusCode, err
			}
		}
	}
}

    Progress Bar - Start

    Till now we have seen how we are updating the progress, Let’s move to see how we will display the progress bar. We are going to print the progress bar every second. First, we will need a ticker to tick for every second. We will use a for-select loop which will read from the ticker and print the progress bar. We also read from a stop channel to stop the progress bar printing.

    Let’s first understand how to print a progress bar on a terminal. The idea is that on every iteration we will move our terminal cursor to the beginning of the first line and print again which will overwrite the previous data.

    We will use “\033[F” which moves the cursor up one line. Based on how many progress bars we are printing we will move up too. When we receive a stop for the progress bar then we finally print the progress bar again without moving the cursor.

    func (sum *summon) startProgressBar(stop chan struct{}) {

	ticker := time.NewTicker(time.Second)
	defer ticker.Stop()

	for {
		select {
		case <-ticker.C:
			for i := 0; i < len(sum.progressBar); i++ {

				sum.RLock()
				p := *sum.progressBar[i]
				sum.RUnlock()

				printProgress(i, p)
			}

			//Move cursor back
			for i := 0; i < len(sum.progressBar); i++ {
				fmt.Print("\033[F")
			}

		case <-stop:
			for i := 0; i < len(sum.progressBar); i++ {
				sum.RLock()
				p := *sum.progressBar[i]
				sum.RUnlock()
				printProgress(i, p)
			}
			return
		}
	}

}

    Progress Bar - Print

    Let’s see how we are printing our progress bar, We are going to use strings.Builder for building our string.

    First, we calculate how much percent of the download is done. We have a PROGRESS_SIZE constant defined which tells us how many characters to print the progress bar on.

    n is the number of characters we need to use from the PROGRESS_SIZE to print the progress. For example, if PROGRESS_SIZE is 20 and we have reached 10 percent download then we need to fill 4 spaces. We loop across the PROGRESS_SIZE constant and print “>” till n and space for the remaining. We are also printing the actual percentage.

    func printProgress(index int, p progress) {

	s := strings.Builder{}

	percent := math.Round((float64(p.curr) / float64(p.total)) * 100)

	n := int((percent / 100) * PROGRESS_SIZE)

	s.WriteString("[")
	for i := 0; i < PROGRESS_SIZE; i++ {
		if i <= n {
			s.WriteString(">")
		} else {
			s.WriteString(" ")
		}
	}
	s.WriteString("]")
	s.WriteString(fmt.Sprintf(" %v%%", percent))

	fmt.Printf("Connection %d  - %s\n", index+1, s.String())
}

    Let’s see our progress bar in action.

    Our progress bar is ready, now we need to handle the interrupt or ctrl+c signals. Once we get it we need to stop the file download and delete the file which we have created.

    Catching Kill / Interrupt Signals

    We will be using the below function to catch signals, We are creating a signal channel of size 1 and then passing it to signal.Notify. Once a kill signal is received we pass it to the stop channel based on the number of chunks we are working with.

    func (sum *summon) catchSignals() {
	sigc := make(chan os.Signal, 1)
	signal.Notify(sigc,
		syscall.SIGHUP,
		syscall.SIGINT,
		syscall.SIGTERM,
		syscall.SIGQUIT)
	go func() {
		s := <-sigc
		for i := 0; i < len(sum.chunks); i++ {
			sum.stop <- fmt.Errorf("got stop signal : %v", s)
		}
	}()
}

    If you remember we did a for-select loop, in which we were reading from the stop channel, this signal will be received there and the reading will be stopped.

    for {
		select {
		case cErr := <-sum.stop:
			return response.StatusCode, cErr

				return response.StatusCode, nil
			}

			if err != nil {
				return response.StatusCode, err
			}
		}
	}

    Let’s see what it looks like when we stop a download in middle. We have used the ctrl+c command to stop and we are able to stop it. This will then remove the file which we had created earlier.
