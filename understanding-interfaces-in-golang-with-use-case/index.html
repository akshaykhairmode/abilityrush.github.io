    Introduction

    If you are like me, you may not have understood the uses of interfaces in Golang. It took me some time but finally, I understood them. Most of my applications did not use interfaces initially but later I started using them wherever possible.

    There are 2  types of interfaces in Golang,

    1. Empty Interface
    2. Named Interface

    Empty Interface

    Empty interface which is represented by interface{} is used when we don’t know the specific data type. For example, when decoding a JSON string that does not have a fixed signature, we use map[string]interface{}, This is because the keys in JSON are always string while the value can be anything.

    If we want to find out the type we can use type assertion or use the reflect package.

    Type assertion converts the interface type to the actual static type.

    For example,

    var i interface{}
    i = 5
    j := i.(int)
    k := i.(int64) //this will fail as 5 is int

    To avoid panic while doing type assertion we can add another variable to the left which is a bool and will tell us if type assertion was successful or not and the program won’t panic.

    k , ok := i.(int64) //this will not panic but the value of ok will be false and k will be an int64 with a value 0

    Named Interface

    Named Interfaces provides us duck typing meaning we can use a named interface as a parameter and all the objects who satisfy the interface methods can be passed to it.

    Let’s implement a stack and create a method that will accept named interface as a parameter.

    Let’s create our interface like below, We can see there are 2 methods, pop, and push. push will push the element to the top of the stack and pop will remove the element from the top of the stack.

    type stacker interface {
          pop() (interface{}, error)
          push(interface{})
    }

    Now let’s create 2 structs that will have a similar signature to this interface. The Data field will store the data we push. We are going to use a dummy interface which will satisfy the interface but will do nothing.

    type stack struct {
          data []interface{}
    }

    type dummyStack struct {
          data []interface{}
    }

    We are now creating methods to attach to stack struct. We can see that these are pointer methods since we want to update the data field we are using pointer. Push is doing a simple append operation on the data and pop is getting the last element and re-slicing the data to remove the last element.

    func (this *stack) push(v interface{}) {
          this.data = append(this.data, v)
    }

    func (this *stack) pop() (interface{}, error) {

         if len(this.data) <= 0 {
                return 0, fmt.Errorf(“Stack is empty”)
          }

          elem := this.data[len(this.data)-1]
          this.data = this.data[:len(this.data)-1]
          return elem, nil
    }

    Let’s create the dummyStack methods, These methods will just print some text and nothing else as we can see below.

    func (this *dummyStack) push(v interface{}) {
          fmt.Println(“Dummy push method called”)
    }

    func (this *dummyStack) pop() (interface{}, error) {

          fmt.Println(“Dummy pop method called”)

          return 0, nil
    }

    Now let’s create our function which accepts the interface. This function will pop 3 elements from the stack. As we can see it accepts the stacker interface as a parameter.

    func pop3Elements(s stacker) {
          for i := 0; i < 3; i++ {
                s.pop()
          }
    }

    Finally, let’s create a stack and dummyStack object and try using the pop3Elements function on it. See main function below, We are pushing few elements and poping them. We are also printing the stack.

    func main() {

          s := &stack{}
          s.push(5)
          s.push(10)
          s.push(15)
          s.push(20)
          fmt.Println(“Stack Before Pop”, s)
          pop3Elements(s)
          fmt.Println(“Stack After Pop”, s)

          ds := &dummyStack{}
          ds.push(50)
          ds.push(100)
          ds.push(“test”)
          pop3Elements(ds)

    }

    The output of the above code will be like below, We can see that the pop has been successful and only one element is remaining after popping. We can also see that the dummy push and pop lines are printed which means that our dummy stack pop function also worked. Full code here

    Stack Before Pop &{[5 10 15 20]}
    Stack After Pop &{[5]}

    Use Case

    Let’s say that we have an application with a log function that writes debug logs to a file. Now we want to make logging configurable. We would need this configurable as writing debug logs in production impacts performance while it is useful in the staging environment.

    We have defined the below types. 

    1. Our constant Environment will come from a config file in production. We are using a constant for an example.
    2. We have defined a logger interface that will have a WriteLog method.
    3. 2 empty structs which are Log and NilLogger. we will attach a method to the log struct which will print a log while NilLogger method will do nothing.
    4. a global LogWriter variable which is of interface type.

    type Logger interface {
          WriteLog(t string)
    }

    type Log struct{}

    type NilLogger struct{}

    var LogWriter Logger

    const (
          Environment = “staging”
    )

    Our WriteLog methods for Log and NilLogger are as below, The method attached to Log is printing while NilLogger is an empty method.

    func (l Log) WriteLog(s string) {
          log.Println(s)
    }

    func (n NilLogger) WriteLog(s string) {

    }

    Our main function code is as below, We can see that based on the Environment variable we are changing our LogWriter. We can assign the structs to LogWriter because we satisfy the interface requirement.

    func main() {

          log.Println(“Started”)

          if Environment == “staging” {
                LogWriter = new(Log)
          } else {
                LogWriter = new(NilLogger)
          }

          LogWriter.WriteLog(“Env is ” + Environment)
    }

    If Environment is staging the output will be as below, Since in the staging environment we want to enable logs the logline should print.

    2009/11/10 23:00:00 Started
    2009/11/10 23:00:00 Env is staging

    Now let’s try with the production environment, We can see that the logline has not been printed because our function is empty for NilLogger.

    2009/11/10 23:00:00 Started

    Conclusion

    We can see the flexibility interfaces provide us, Interfaces are very powerful when used correctly. This is it for interfaces.
