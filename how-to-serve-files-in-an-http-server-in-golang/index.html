    • How to serve files in an HTTP server in Golang

    Introduction

    Hey friends, Today we will see how we can serve a file on our HTTP server that clients can download.

    Let’s see the 2 ways we can do it,

    1. In Memory – Load the file in memory and serve from it. (Can be used for small files like configurations)
    2. Buffering – Open the file on every request and serve it. (Used for large files which could cause memory issues with method 1)

    HTTP Server

    We will use a very simple HTTP server with minimum configuration and 2 routes. Below is the server and handler code.

    func main() {

    //....

    server := http.Server{
    		Handler: http.HandlerFunc(handle),
    		Addr:    ":9000",
    	}
    
    log.Println("starting server")
    log.Println(server.ListenAndServe())

}

func handle(w http.ResponseWriter, r *http.Request) {

	log.Println("path", r.URL.Path)

	switch r.URL.Path {
	case "/download-small-file":
		handleSmallFile(w, r)
	case "/download-large-file":
		handleLargeFile(w, r)
	default:
		w.WriteHeader(http.StatusNotFound)
		w.Write(NotFound)
	}

}

    HTTP Headers

    We need to write some specific headers to the client so that they can know we are outputting a file. You can read the inline comments below.

    func setHeaders(w http.ResponseWriter, name, len string) {
	//Represents binary file
	w.Header().Set("Content-Type", "application/octet-stream")
	//Tells client what filename should be used.
	w.Header().Set("Content-Disposition", fmt.Sprintf(`attachment; filename="%s"`, name))
	//The length of the data.
	w.Header().Set("Content-Length", len)
	//No cache headers.
	w.Header().Set("Cache-Control", "private")
	//No cache headers.
	w.Header().Set("Pragma", "private")
	//No cache headers.
	w.Header().Set("Expires", "Mon, 26 Jul 1997 05:00:00 GMT")
}

    Our Handlers

    Let’s start with our buffered handler. Here we will simply open the file, get the length of the contents and use the io.Copy function from the IO package which copies the data in a buffered manner.

    Below is the handler code

    func handleLargeFile(w http.ResponseWriter, r *http.Request) {
	//Open file
	f, err := os.Open(LargeFileName)
	if err != nil {
		handleError(err, w)
		return
	}
	defer f.Close()

	//read the file info
	info, err := f.Stat()
	if err != nil {
		handleError(err, w)
		return
	}

	//Set the headers
	setHeaders(w, LargeFileName, strconv.Itoa(int(info.Size())))
	w.WriteHeader(http.StatusOK)

	//Copy without loading everything in memory
	n, err := io.Copy(w, f)
	if err != nil {
		handleError(err, w)
		return
	}
	log.Printf("LARGE :: Written : %d", n)
}

    Next, for the in-memory handler, we will read the file once our server starts and keep sending the same bytes. Quite straightforward. Code below,

    
func main() {
    
    //...
    
    f, err := os.Open(SmallFileName)
    if err != nil {
    	panic(err)
    }
    
    smallFile, err = io.ReadAll(f)
    if err != nil {
    	panic(err)
    }
    
    //...

}

func handleSmallFile(w http.ResponseWriter, r *http.Request) {
	setHeaders(w, SmallFileName, strconv.Itoa(len(smallFile)))
	n, err := w.Write(smallFile)
	if err != nil {
		handleError(err, w)
		return
	}
	log.Printf("SMALL :: Written : %d", n)
}

    We are running our server and using wget to download the file. We can see in the below image that we are successfully downloading the files. We are using –content-disposition so that wget uses the file name coming from the server.

    Conclusion

    To Conclude,

    We saw how we can serve a file by sending appropriate headers. We also saw how we can let clients download a large file.

    (NOTE:: since for large handlers we are opening the file, if there are too many download requests, we may reach the OS file descriptor limit. So we need to take care of that)
