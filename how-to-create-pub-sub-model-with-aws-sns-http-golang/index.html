    Introduction

    Hey folks, today we will see how we can work on a Pub Sub model using AWS SNS as our streaming service, how to publish messages to SNS and how to subscribe to the topic. We are going to use the HTTP method to subscribe.

    In simple terms, a Pub-Sub model is a state in which there is one publisher and there are one or more than one subscribers who receive the same message.

    An example would be the radio towers, they are publishing on a radio frequency and anyone owning a radio can subscribe to the frequency and get the same message or audio stream.

    We discussed AWS free tier account and ID and Secret in our AWS S3 post.

    SNS Subscribers

    So SNS supports multiple subscribers to the topic. The subscribers currently available are,

    1. Amazon Kinesis
    2. Amazon SQS
    3. AWS Lambda
    4. Email
    5. HTTP/HTTPS
    6. SMS
    7. Platform Application Endpoint.

    For more information, you can see this link.

    Let’s see how the HTTP method will work,

    1. We will first create a topic in SNS.
    2. We will create an HTTP web server with only a base path.
    3. We will call subscribe from our code to our base path.
    4. SNS will send a confirmation event on our base path with some token information.
    5.  We will then use that token to call the confirmation API of SNS and the subscriber will get enabled.
    6. We will also keep publishing some messages in another goroutine.

    Creating Topic In SNS

    We are creating the topic from the AWS Console.  Our topic name will be test-topic.

    HTTP server

    The event which SNS sends has around 10 keys. Below are the keys relevant for us for now so we have created a struct based on that.

    RequestType  – We will receive SubscriptionConfirmation in value when it’s a confirmation request.
    Message – Contains the actual message we send.
    SubscribeURL – Contains the URL with Token and TopicARN. This is required for confirming the subscription.

    type SNSEvent struct {
	RequestType  *string `json:"Type"`
	SubscribeURL *string `json:"SubscribeURL"`
	Message      *string `json:"Message"`
}

    We are going to create an HTTP server with only the root path. Below is our code,

    func startHttpServer(wg *sync.WaitGroup, port string) {
	defer wg.Done()
	server := http.Server{
		Addr:    port,
		Handler: handler(),
	}
	log.Println("starting server on port 8000")
	err := server.ListenAndServe()
	log.Fatal(err)
}

    Now we see our handler code, It’s quite straightforward. After decoding the request body, we check if the request is of type confirmation, if yes then we will call the confirmation API or else print the message.

    func handler() http.HandlerFunc {
	return func(w http.ResponseWriter, r *http.Request) {
		body, err := io.ReadAll(r.Body)
		if err != nil {
			log.Println(err)
		}

		req := SNSEvent{}

		if err := json.Unmarshal(body, &req); err != nil {
			log.Println(err)
			w.WriteHeader(500)
			return
		}

		//confirmation request
		if *req.RequestType == "SubscriptionConfirmation" {
			confirm(req)
			return
		}

		//messages
		log.Println("Got Message", *req.Message)
		w.WriteHeader(200)
	}
}

    Subscribe Code

    Let’s see how we can subscribe to the topic from our code. First, we create the AWS session and SNS service objects and set them in global variables. In production code, we should not use global variables.

    Here, I have already set the AWS ID and SECRET in my environment so I don’t need to pass it into the code.

    var sess = session.Must(session.NewSession(
	&aws.Config{
		Region: aws.String("ap-south-1"),
	},
))

var snsService = sns.New(sess)

//subscribe subscribes to the topic. This can be done from the AWS console also.
func subscribe(httpURL string) {
	inp := &sns.SubscribeInput{
		Endpoint: aws.String(httpURL),
		Protocol: aws.String("http"),
		TopicArn: aws.String("arn:aws:sns:ap-south-1:XXXXXXXXXX:test-topic"),
	}

	output, err := snsService.Subscribe(inp)
	if err != nil {
		log.Println(err)
	}

	log.Println("Output", output.String())
}

    Confirmation Code

    In our confirmation code, we are reading the event, finding the token & topic ARN, and passing it to AWS again.

    func confirm(request SNSEvent) {

	vals, err := url.ParseQuery(*request.SubscribeURL)
	if err != nil {
		log.Println(err)
	}

	token := vals.Get("Token")
	topicARN := vals.Get("TopicArn")

	confirm := &sns.ConfirmSubscriptionInput{
		Token:    aws.String(token),
		TopicArn: aws.String(topicARN),
	}
	output, err := snsService.ConfirmSubscription(confirm)
	if err != nil {
		log.Println(err)
	}

	log.Println("confirm output", output.String())
}

    Publisher

    Our publisher will keep publishing messages every 2 seconds. Below is the code,

    func publish() {
	for {
		input := &sns.PublishInput{
			Message:  aws.String("Current time is " + time.Now().String()),
			TopicArn: aws.String("arn:aws:sns:ap-south-1:XXXXXXXXXXX:test-topic"),
		}
		//We are ignoring the output here. Its the messageid.
		_, err := snsService.Publish(input)
		if err != nil {
			log.Println("publish error", err)
		}

		time.Sleep(2 * time.Second)
	}
}

    Main Function

    Let’s see what our main function looks like, We are using flags to get the public endpoint that we will provide to SNS. I am using ngrok to expose my web server on the internet.

    var publicHttpEndpoint string

func main() {

	flag.StringVar(&publicHttpEndpoint, "endpoint", "", "-endpoint")
	flag.Parse()

	if publicHttpEndpoint == "" {
		log.Fatal("Public Endpoint is required")
		flag.PrintDefaults()
	}

	wg := &sync.WaitGroup{}
	wg.Add(1)
	go startHttpServer(wg, ":8000")
	subscribe(publicHttpEndpoint)
	go publish()

	wg.Wait()
}

    Output

    First, let’s start the ngrok service to expose port 8000 over the internet. We get the HTTP URL and pass it to our application. We have already hardcoded the topic ARN so we don’t need to pass it.

    Let’s run our binary and see the output. We will use this command to run it. go run main.go -endpoint http://e4ee-2405-201-31-30fe-ed83-4150-334f-fadb.ngrok.io

    Conclusion

    As we can see in the output, We are successfully publishing and also getting events on our HTTP Web server exposed via ngrok.
